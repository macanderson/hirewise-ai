## üñºÔ∏è Shared Brand Logo

- Place your brand logo and other shared assets in the `/assets/` directory at the root of the monorepo.
- Example: `/assets/logo.png`

### Using the Logo in Web Apps

Symlink the logo into your app's public directory:

```sh
ln -s ../../assets/logo.png apps/web/public/logo.png
```

Now you can reference it in your frontend as `/logo.png`.

## üóÑÔ∏è Database Schema & Prisma

### Location

- The Prisma schema and migrations are in `apps/api/prisma/`.
- Example:
  - `apps/api/prisma/schema.prisma`
  - `apps/api/prisma/migrations/`

### Generating the Python Client

- Uses [`prisma-client-py`](https://github.com/RobertCraigie/prisma-client-py).
- Output is configured in `schema.prisma`:

  ```prisma
  generator client_py {
    provider = "prisma-client-py"
    output   = "./../database/client"
    ...
  }
  ```

- Generate the client:
  ```sh
  cd apps/api
  prisma py generate --schema=prisma/schema.prisma --output=database/client
  ```
- Import in Python:
  ```python
  from prisma_client import Prisma
  ```

### Generating TypeScript Types for Frontend

- (Optional) Use Prisma or a tool like Kanel/pg-to-ts to generate types from your schema.
- Recommended: Output to a shared package (e.g., `packages/db-types/`) or directly into your frontend app (e.g., `apps/web/types/db/`).
- Example Prisma generator block (uncomment and adjust as needed):
  ```prisma
  generator client_ts {
    provider = "prisma-client-js"
    output   = "../../apps/web/types/db"
    ...
  }
  ```
- Generate types:
  ```sh
  cd apps/api
  npx prisma generate --schema=prisma/schema.prisma
  ```
- Import types in your frontend:
  ```ts
  import { User, Project } from './types/db';
  ```

## üèóÔ∏è Continuous Integration (CI)

The monorepo uses GitHub Actions for CI, defined in `.github/workflows/ci.yaml`.

### Workflow Steps

1. **Checkout code**
2. **Setup Node.js and pnpm**
3. **Cache pnpm store**
4. **Install dependencies**
5. **Run linting**
6. **Run type checking**
7. **Run tests**
8. **Build all apps/packages**

```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Install pnpm
        uses: pnpm/action-setup@v2
        with:
          version: 8
          run_install: false
      - name: Get pnpm store directory
        id: pnpm-cache
        run: echo "store_path=$(pnpm store path --silent)" >> $GITHUB_OUTPUT
      - name: Setup pnpm cache
        uses: actions/cache@v3
        with:
          path: ${{ steps.pnpm-cache.outputs.store_path }}
          key: ${{ runner.os }}-pnpm-store-${{ hashFiles('**/pnpm-lock.yaml') }}
          restore-keys: |
            ${{ runner.os }}-pnpm-store-
      - name: Install dependencies
        run: pnpm install
      - name: Run linting
        run: pnpm lint
      - name: Run type check
        run: pnpm type-check
      - name: Run tests
        run: pnpm test
      - name: Build
        run: pnpm build
```
